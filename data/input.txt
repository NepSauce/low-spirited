Machine learning is the process of creating systems that improve their behavior by learning patterns from data instead of relying entirely on manually written rules.

Traditional programming works like this:

Input + Rules -> Output

Machine learning works like this:

Input + Correct Outputs -> Learned Rules

A machine learning model is a mathematical function with adjustable parameters.

Training is the process of adjusting those parameters so the model produces more accurate outputs.

A dataset is a collection of examples used during training.

Each example usually contains:
- input data
- expected output data

Example:

Input:
"The sky is blue"

Expected Output:
"blue"

The model learns statistical relationships between tokens, words, or features.

A parameter is a numerical value inside the model that changes during learning.

Modern neural networks often contain millions or billions of parameters.

A neural network is composed of layers.

Each layer transforms data into another representation.

Early layers may learn simple patterns.

Deeper layers may learn abstract relationships.

A tensor is a multidimensional numerical array.

Examples:

Scalar:
5

Vector:
[1, 2, 3]

Matrix:
[
  [1, 2],
  [3, 4]
]

3D Tensor:
A stack of matrices.

PyTorch represents nearly everything as tensors.

Machine learning models operate primarily through tensor operations.

An embedding is a learned vector representation of a token.

Words with similar meanings may develop similar embeddings.

Example:

king -> [0.1, 0.8, ...]
queen -> [0.12, 0.79, ...]

Distance in vector space can represent semantic similarity.

Natural language processing focuses on understanding and generating human language.

A tokenizer converts raw text into tokens.

Tokens may represent:
- characters
- words
- subwords
- symbols

Character tokenizer example:

"cat" -> ['c', 'a', 't']

Subword tokenizer example:

"running" -> ["run", "ning"]

A vocabulary is the complete set of known tokens.

Each token is mapped to a unique integer ID.

Example:

{
  "hello": 0,
  "world": 1
}

Encoded text:
"hello world" -> [0, 1]

Language models predict the next token in a sequence.

Example:

Input:
"The computer is"

Prediction:
"running"

This process is called autoregressive generation.

The model repeatedly predicts one token at a time.

Transformer models process tokens using self-attention.

Self-attention allows tokens to examine relationships with previous tokens.

Example:

"The animal did not cross the street because it was tired."

The word "it" depends on understanding previous words.

Attention helps the model determine which earlier tokens matter most.

Attention produces weighted relationships between tokens.

Important tokens receive stronger attention scores.

Transformers process sequences in parallel during training.

This made them significantly more efficient than older recurrent architectures.

A transformer block commonly contains:
- self-attention
- feed-forward network
- residual connection
- layer normalization

Residual connections help gradients flow through deep networks.

Layer normalization stabilizes activations during training.

The feed-forward network applies nonlinear transformations to each token representation.

Training uses a loss function.

Loss measures prediction error.

Example:
- low loss means accurate predictions
- high loss means inaccurate predictions

Cross entropy loss is commonly used for language models.

Optimization algorithms reduce loss over time.

Gradient descent adjusts parameters in the direction that reduces error.

Backpropagation computes gradients through the chain rule from calculus.

A gradient measures how much a parameter influences the loss.

The learning rate controls update size during optimization.

Too small:
- training becomes slow

Too large:
- training becomes unstable

Epoch:
One full pass through the dataset.

Batch:
A small subset of training examples processed together.

GPUs accelerate matrix and tensor operations through parallel computation.

Overfitting occurs when a model memorizes training data instead of learning general patterns.

Underfitting occurs when the model is too weak to capture meaningful patterns.

Generalization is the ability to perform well on unseen data.

The term "deep" refers to network depth.

Depth allows models to learn hierarchical representations.

Example:

- middle layers may detect words

Parameters are adjusted during training through optimization.

A model begins with random parameter values.

Initially, predictions are nearly meaningless.

Training gradually shapes parameter values into useful statistical structures.

Neural networks are function approximators.

They learn mappings between inputs and outputs.

A neuron computes:
- weighted sum
- bias addition
- nonlinear activation

Activation functions introduce nonlinearity.

Without nonlinearity, deep networks collapse into simple linear transformations.

Common activation functions:
- ReLU
- GELU
- Sigmoid
- Tanh

ReLU:
f(x) = max(0, x)

GELU is commonly used in transformers.

The forward pass computes predictions from input data.

The backward pass computes gradients for parameter updates.

Backpropagation propagates error information backward through the network.

The chain rule from calculus enables gradient computation across many layers.

Vanishing gradients occur when gradients become extremely small in deep networks.

Exploding gradients occur when gradients become excessively large.

Modern architectures use normalization and residual connections to stabilize training.

A residual connection adds earlier activations back into later outputs.

This helps preserve information flow across deep networks.

Optimization landscapes contain many local minima and saddle points.

Training attempts to move parameters toward lower-loss regions.

Stochastic gradient descent estimates gradients using random mini-batches.

Adam optimizer improves training stability through adaptive learning rates.

Language models learn probability distributions over token sequences.

Example:

P("world" | "hello")

The model estimates how likely a token is given previous context.

Softmax converts logits into probabilities.

Logits are unnormalized prediction scores.

Softmax ensures probabilities sum to 1.

Temperature controls randomness during generation.

Low temperature:
- deterministic
- repetitive
- conservative outputs

High temperature:
- diverse
- creative
- unstable outputs

Sampling strategies include:
- greedy decoding
- top-k sampling
- nucleus sampling

Greedy decoding always chooses the highest-probability token.

Top-k sampling limits choices to the top k candidates.

Nucleus sampling selects from a probability mass threshold.
